HCIA-OSI、子网划分

发表于 2021-05-17 | 分类于 HCIA |

OSI模型

  • IOS

国际标准化组织（International Organization for Standardization，简称为ISO）是标准化领域中的一个国际性非政府组织。ISO一来源于希腊语“ISOS”，即“EQUAL”——平等之意。ISO成立于1947年，是全球最大最权威的国际标准化组织，全体大会是ISO最高权力机构，理事会是ISO重要决策机构，中国是ISO常任理事国。 [6] 中国是ISO的正式成员，代表中国参加ISO的国家机构是中国国家标准化管理委员会（由国家市场监督管理总局管理）。

  • OSI

意为开放式系统互联。国际标准组织（国际标准化组织）制定了OSI（Open System Interconnection）模型。这个模型把网络通信的工作分为7层,分别是物理层,数据链路层,网络层,传输层,会话层,表示层和应用层。1至4层被认为是低层，这些层与数据移动密切相关。5至7层是高层，包含应用程序级的数据。每一层负责一项具体的工作，然后把数据传送到下一层。

  • 控制层面

    7应用层、6表示层、5会话

  • 数据层面

    4==传输层==、3==网络层==、2==数据链路层==、1物理层

分层的思想

上三层：

思科默认这三层没有PUD，华为有，如7层APUD

  • 应用层：接收用户的数据，人机交互的接口，面向的应用程序。

  • 表示层：将逻辑语言（软件语言）转换为机器语言（二进制语言），翻译 ，加密

  • 会话层：针对传输的每一种数据（流量）建立（管理：维持、终止）一条==虚连接==（为了防止不同类型的数据互相影响）

传输层

PDU： 协议数据单元（数据在不同层的表示方式）

==4层PDU：segment 分段 分片==

  • 作用：1.区分流量（端口号） 2.定义数据传输方式（TCP/UDP）

  • 端口号： port ID , 数值范围1-65535

  • 静态端口：又称为注明端口范围1-1023

  • 动态端口：范围1024-65535

常见端口号

HTTP 80 TCP
HTTPS 443 TCP
RIP 520 UDP
Telnet 23 TCP 远程登录服务
SSH 22 TCP 远程登录（安全性较高 加密 认证）
DNS 53 TCP和UDP 域名解析

TCP(可靠传输)/UDP(不可靠传输)

可靠：把信息一定要传递给对方；不可靠：只管传输。

TCP：传输控制协议，是一种面向连接的可靠传输协议

UDP：用户数据报协议，是一种非面向连接的不可靠的传输协议

  • 不可靠传输方式流量特点（具备一下特点时使用UDP）
    • 大流量
    • 实时性较高
    • 对数据丢失不敏感
  • 可靠传输
    • 特点：1.确保数据一定传输给对方
    • 如何保证可靠性： 1.确认机制 2.重传输机制
  • 三次握手机制： 保证了面向连接 ，又称为TCP的三次握手
  • 确认机制：显式确认 隐式确认
    • 显式确认：有明确回复的
    • 隐式确认：没有明确的回复（没有回复默认收到）
  • 优化机制：
    • 流控机制（滑动窗口机制）通俗易懂讲解TCP流量控制机制，了解一下 - 帅地 - 博客园 (cnblogs.com)
    • 重排序
  • 数据的分段传输： 传输较大数据时按照MTU值进行分段传输。
  • ==MTU==： 最大传输单元，默认为1500字节，可以修改（但不建议）。

TCP三次握手

TCP三次握手如图：

第一次握手

客户主动（active open）去connect（/访问）服务器，并且发送SYN request（同步请求）假设序列号为J,服务器是被动打开(passive open)

第二次握手

服务器在收到SYN后，它会发送一个SYN以及一个ACK（应答）给客户，ACK的序列号是 J+1表示是给SYN J的应答，新发送的SYN K 序列号是K

  • 这里的SYN确认客户端的端口是同步状态
第三次握手

客户在收到新SYN K, ACK J+1 后，也回应ACK K+1 （隐式确认）以表示收到了，然后两边就可以开始数据发送数据了

网络层

==三层PDU：packet 包==

编址 寻址

地址

编址协议： IP （IPV4 IPV6） IPX apple talk novell

编址协议定义地址

  • IPv4：采用32个二进制编址， 一个二进制 称为1位 bit 。

  • IP地址：32个二进制，0和1构成 用于编址

  • 网络掩码：32个二进制，连续的1+连续的0构成，连续的1代表着网络位，连续的0代表主机位。

​

数据单位

Bit：比特 ，一个二进制 1000bit=1Kbit 1000kbit=1Mbit 1000Mbit=1Gbit

Byte：字节，八个bit 1Byte=8bit 1000byte=1K 1000K=1M

100M=100Mbps （b=bit）

100Mbps=12.5MBps 承诺突发

地址书写

书写一个完整的IP地址时 ：IP地址+网络掩码

地址分类：

A类地址： 第一位固定为0

0XXX XXXX —0-127（1-126），网络掩码默认为 255.0.0.0
有2^7^个网段

B类地址：前两位固定为10

10XX XXXX—128-191，网络掩码默认为255.255.0.0
有2^14^个网段，看网络位，网络位有16位，减去2个固定的，剩下14个

C类地址：前三位固定110

110X XXXX—192-223，网络掩码默认为255.255.255.0
2^21^个网段

D类地址：前四位固定为1110

1110 XXXX—224-239，组播地址，无掩码

E类地址：前四位固定为1111

1111 XXXX—240-255，科研地址。

特殊地址

  • 0.X.X.X 无效地址（保留地址），0.0.0.0 无效地址 占位

  • 127.0.0.1 本地测试 （127.X.X.X 测试地址）

  • 网络号，网络位不变 主机位全为0 的地址（==描述一个网段==）

    • 162.1.1.1-162.1.0.0

    • 255.255.0.0

  • 受限广播地址（受限到自己的网段）

    • 255.255.255.255
    • 代表所有地址的地址，但是只给自己网段内的用户发送

  • 定向（直接）广播地址，网络位不变，主机位全为1

    ==200.1.1==.1>>>==200.1.1==.255

    ==255.255.255==.0

  • ==本地链路地址：link-local { 169.254.0.0 255.255.0.0 }==，微软买的

在配地址的时候，主机位全为0以及主机位全为1的地址不能分配

A B C 类地址称为单播地址

单播：一对一的传输方式

组播：一对多的传输方式

广播：一对所有传输方式

私有地址

==二层PDU ：frame 帧==

局域网（以太网 Ethernet）

广域网 （PPP HDLC ATM(异步传输网络) FR 以上为协议）

  • 二层地址：Mac地址（介质访问控制）—物理地址 硬件地址 烧录(BIA)地址

  • LLC层：逻辑(L)链路(L)控制(C)子层（描述上层使用什么协议:ipv4/ipv6…）

  • MAC层：介质（媒体）访问控制子层（描述二层地址）

物理层

==一层PDU：bit 比特流==

网络硬件的机械特性 光学特性 电学特性

传输介质：光纤 、双绞线（RJ45）

一层各种线和设备

双绞线：红色线负责端口转发，其他不起作用（千兆以下，其他的可用来做电话线（两根一组））

按照屏蔽分类：屏蔽双绞线 （STP，也会屏蔽自身磁场，所以传输距离弱于非屏蔽线） ；非屏蔽双绞线（UTP）//屏蔽强电，强电有电磁场，但是自身的弱电也会屏蔽

按照拧度进行分类：3类 4类—（10Mbps） 5类 超5类(5类的屏蔽)—（100Mbps） 6类 超6类(6类的屏蔽) （1000Mbps–在6类线中8根同时使 用）

  • 568A线序（AT&T公司）：绿白、绿、橙白、蓝、蓝白、橙、棕白、棕

  • 568B线序（IBM公司）：==橙白（发1）、橙（发2）、绿白（收1）、蓝、蓝白、绿（收2）、棕白、棕==

平行线（直通线）：又称为直通线 ，线序相同。不同层设备使用平行线。

交叉线：（同层设备使用）：线序不同（一头568A 、 一头568B）。同层设备使用交叉线。

同层设备如果使用平行线，这会造成一头发送另一头也发送，一头接收另一头也接收。

全反线：又称为console线，配置线，线序相反，用于用户控制网络设备。

路由器：三层设备（网络层），识别IP地址。不同的接口在不同的网段（广播域）

交换机（网桥）：二层（数据链路层）设备，使用MAC地址，所有的接口默认在同一个广播域，一个接口是一个冲突域。

集线器（hub）：放大器 放大信号，属于1层（物理层）设备，识别bit流，hub上所有的接口都在同一个冲突域中。

也可以用于长距离时的信号放大。
